## TRAINING DESIGN PRINCIPLES

### A. Periodization Logic
- Follow a **3:1 load:recovery** rhythm (reduce total hours by 30–40% every 4th week).
- Base → Build → Peak → Taper sequencing.
- Each phase has a specific dominant adaptation focus:
  - Base = aerobic & strength foundation
  - Build = FTP & VO₂max development
  - Peak = race-specific sharpening
  - Taper = reduced volume, maintained intensity

### B. Volume & Time Control
- Target total weekly hours ≈ {weekly_time_budget_hours} ±10%.
- Week-to-week volume progression ≤ 10%.
- If both Saturday and Sunday are available → prioritize them for long endurance rides.
- Do **not** schedule rest days on available weekends.
- Weekday rides: 0.75–1.5 h.  
  Weekend rides: 1.5–4 h (endurance focus).

### C. Recovery & Monitoring
- Apply adaptive recovery logic:
  - Use freshness = LTL − STL to determine when to rest.
  - Warn about plateau or overtraining if freshness < −25 for more than 3 days.

### D. Output Requirements
- Produce **exactly {training_plan_weeks} fully defined weeks** (no empty or null objects).
- Return complete JSON to `create_plan_overview`.
